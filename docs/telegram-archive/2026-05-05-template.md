# Telegram Morning Briefing — Archive of 2026-05-05

This is the morning Telegram delivery as it stood the day we decided to rebuild it. Captured for two reasons:

1. So we never silently drift away from what existed (good ideas can be lost in a rewrite).
2. So the rebuild can be measured against this — keep the parts that worked, drop the parts that didn't.

The system on this date had **multiple independent agents** delivering messages directly to the operator, with no synthesis layer that actually worked. The Telegraph (the Herald) was supposed to be the one daily paper, but it wasn't reading its inputs and was silently lying.

## What we want to keep (in spirit)

The **shape** of what we had isn't all bad — it just wasn't connected up:

- Single morning paper (the Telegraph idea is right)
- Sections: Front Page (urgent) / Today's Work / Long Read / Library / Classifieds — this was the design, never delivered
- Three editions for three audiences: Daily (king), Brief (Barry, technical light), Weekly (subjects, friendly) — see TODO.md b0868249
- Specialised agents writing to a shared bus, Herald synthesising

## What we want to drop

- Multiple Telegram messages in the morning. **One message per intended audience, delivered once.**
- Two parallel registries (admin-center `apps` vs Steward `VILLAGES` dict).
- The n8n delivery path duplicating what the Herald should produce.
- Side-effect side-channel sends (Herald triggering Hand which also sends).
- Any "all clear" message that doesn't prove its inputs were collected — silence-on-failure is the worst failure mode.

## The contract for the rebuild

Whatever replaces this must satisfy:

1. **One source of truth for the registry.** Every system the kingdom watches lives in one place. Adding/removing systems is one operation.
2. **One delivery per edition.** The king receives at most one message per scheduled edition. Duplicates are bugs.
3. **Failure must be loud.** If a council member's input couldn't be collected, the Telegraph says so explicitly. "All services healthy" must mean every input was gathered AND every input said healthy.
4. **Production services are checked from one probe.** Never two agents holding two opinions of the same system.
5. **Truth before pretty.** The synthesis layer can't lie by omission.

These are non-negotiable for the rebuild.

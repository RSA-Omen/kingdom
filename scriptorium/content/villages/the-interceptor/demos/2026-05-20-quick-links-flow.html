Quick Links routing — data flow

When a user clicks the Quick Links button inside Pronto Xi, the request lands at The Interceptor. Matched URLs are relayed to NextCloud; everything else passes through to Pronto's own URL handler unchanged.

Interceptor path / matched URL
Pass-through / non-matched

Notes

  • The Interceptor does not modify the URL beyond issuing a redirect for matched requests.
  • It holds no state across requests — there is no database and no event stream.
  • Quick Links is the only entry point that matters. Anything that doesn't come from Quick Links never reaches the matching logic.

# fix(receipts): stop misidentifying \$ as ZAR; default display currency to \$

OCR engines (tesseract, glm-ocr) routinely read \$ as bare R, causing the
currency detector to flag USD receipts as South African Rand.

- Backend: require explicit 'ZAR' word for ZAR detection; bare 'R' before
  digits is no longer accepted as a currency indicator
- Frontend: change formatCurrencyAmount fallback from 'R' to '\$' so receipts
  with no detected currency display as dollars instead of rand

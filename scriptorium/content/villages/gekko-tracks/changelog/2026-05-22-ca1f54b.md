# fix(cardholder): wire project/cost picker into mobile transaction-detail modal

On mobile, tapping "Open in modal" on a batch card revealed the Transaction
Details modal with Project and Cost Category shown as plain read-only text.
Desktop never hit this path because edits happen inline in the table.

- Hoist canEditProjectCost so it's in scope for both render branches
  (ledger view stays inactive because selectedBatchId === null there)
- Make Project and Cost Category rows in the detail modal tappable to
  launch the existing projectCostPickerModal, mirroring the mobile card
  taps already wired at lines 5519 / 5546
- Use 44px touch targets and italic placeholder text for empty fields
- Skip the picker for split transactions (splits own their own project/cost)

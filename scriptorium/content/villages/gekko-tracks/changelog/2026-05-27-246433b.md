# feat(notifications): Telegram batch-summary alert + email log API

- notification_service: debounced 90-second batch summary — resets
  each time a cardholder batch is assigned; fires once after Finance
  finishes releasing to all cardholders with sent count + missing list
- telegram_notifier: send_batch_summary_alert() — one clean message
  per batch release (✅ all sent / ⚠️ N of M with missing names)
- notification_service: Eloise manager-approved emails now logged to
  notification_logs (previously sent but never recorded)
- notifications router: GET /api/notifications/logs (filterable) +
  GET /api/notifications/monthly-cadence (cardholder × month grid)
  — consumed by Kingdom dashboard Email Logs page

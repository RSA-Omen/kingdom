Bender call — sequence diagram

A typical Bender call has three actors: the Caller, Bender itself, and Pronto Xi. The Caller never talks to Pronto directly — Bender opens a session, plays keystrokes, scrapes the result, and hands JSON back.

The five steps

  1. Request: Caller hits Bender with the script name, parameters, and which credential profile to use.
  2. Session: Bender opens (or reuses) a Pronto session under that profile.
  3. Playback: Bender plays the recorded keystroke sequence for the script.
  4. Scrape: Bender reads the relevant fields from Pronto's screen and parses them.
  5. Return: Bender hands structured JSON back to the Caller. The session stays warm for the next call.

Bender — data flow

Bender is the only path between Kingdom callers and Pronto Xi for everything beyond URL routing. Callers send requests, Bender translates them into Pronto UI actions, and structured results flow back.

Key points

  • One-way for requests, one-way for results. Callers never reach Pronto; Pronto never directly answers a caller.
  • Bender owns no data. Everything it surfaces is read from Pronto. Anything it writes lands in Pronto.
  • Sessions are per credential profile. Two scripts under the same profile share a session; two profiles run independently.
  • [[ap-process]] is the heaviest current caller — it consumes Bender's supplier master export (3,184 suppliers in the recent dump).

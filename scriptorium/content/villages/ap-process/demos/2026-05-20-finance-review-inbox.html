Finance review inbox — user interaction

Where a Finance reviewer spends time. Each invoice the pipeline could not auto-close lands here with the candidate matches already surfaced and one primary action visible inline.

What the reviewer does

  • One primary action per row. The matcher decides what's most likely useful — Confirm, Accept, Attach, or Add. The reviewer either approves or picks a different path.
  • Confidence shown inline. Green for high (≥90%), amber for low. The reviewer never has to dig for it.
  • Approved rows leave the inbox immediately and queue for posting to Pronto via [[bender]].
  • The pipeline never asks "what should I do here?" — it only surfaces the cases it can't close, with the data the reviewer needs to decide.

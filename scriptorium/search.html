The Interceptor process / Routing rules / EOM behavior villages/the-interceptor#eom

H2 End-of-month routing & the reconciliation queue

During the EOM window (23:00–23:45 SAST on the last day), Interceptor holds traffic destined for SERV_ORD_CLS in a per-village queue and replays it once the lock clears. This is what produces the visible reconciliation lag in Gekko Tracks and the late-night bursts in the Bender run log.

H3 Dispatch artifacts & cross-village reconciliation

Closure attachments — signed POD, photos, invoice PDFs — are indexed against the originating job.id so the reconciliation step in Tracks can confirm a complete artifact set before flipping the job to closed. Missing artifacts hold the job in pending_reconciliation.

Gekko Tracks app / Runbook / Stuck closure villages/gekko-tracks/runbook.md#stuck-closure

H3 Job stuck in pending_reconciliation

Symptoms: job has been on-site + signed off for > 30 min but has not flipped to closed. Check the Bender run log for the run keyed by job.id. If the run completed with state: "ok" but the reconciliation field is still pending, the Pronto webhook is the most likely culprit — replay it via the runbook script.
